StarClaimManager.js

Manages the barcode reader and magstripe reader, line display of the claim by StarWebPRNT request element and StarWebPRNT response element. Sends StarWebPRNT request element which is with the information of the claim and checks status from received StarWebPRNT response element.

Constructor

NameDescription
StarClaimManagerCreates StarClaimManager object.

Method

NameDescription
sendMessageAdd the settings for Claim management and send Sends StarWebPRNT request element with the settings.
isBCRClaimedCheck the Claim status of barcode reader device
isMSRClaimedCheck the Claim status of magstripe reader device
isLDClaimedCheck the Claim status of line display device
getBCRDeviceNameCapabilityGet the name of barcode reader device
getBCRModeCapabilityGet the supported scan modes of barcode reader device
getBCRSymbolCapabilityGet the supported barcode symbol of barcode reader device
setPollingIntervalSet the sending interval time for polling of Claim management(*1)
setBCREventListenerSet the callback method which is related with barcode reader device
setMSREventListenerSet the callback method which is related with magstripe reader device
setLDEventListenerSet the callback method which is related with line display device
removeBCREventListenerDelete the callback method which is related with barcode reader device
removeMSREventListenerDelete the callback method which is related with magstripe reader device
removeLDEventListenerDelete the callback method which is related with line display device

(*1) The polling for keeping the claim status by current using browser
       is started when the Claim start is success(*2) after sending the request elements
       which is included the "claim='true'" in the first time.
       And The polling is finished when the Claim finish is success(*3) after sending
       the request elements which is included the "claim='false'".
       The polling start and finish are automatically controled by the sendMessage method of StarClaimManager.js.
(*2) Character string which is included in the response elements to receive at the time of Claim start success
       "<bcr><claim>true</claim></bcr>"
(*3) Character string which is included in the response elements to receive at the time of Claim finish success
       "<bcr><claim>false</claim></bcr>"

getBCRModeCapability

Get the supported scan modes of barcode reader device.

syntax

getBCRModeCapability();

Parameter

NameDescriptionObject typeOmission
----

Return value

DescriptionObject type
List of support situation for the scan mode
*Return value is following object which has a property
[Property]
  • OFF … Turn off<
  • SCAN_ONCE … The scan mode of once reading
  • AUTO … The scan mode of continuous reading with sights
  • CONTINUOUS … The scan mode of continuous reading without sights
  • MOBILE_PHONE … The scan mode of one time reading for LCD screens
  • MOBILE_CONTINUOUS … The scan mode of continuous reading for LCD screens
[Value of property (Boolean)]
  • true … This mode is supported
  • false … This mode is not supported.
Object
Example

    if (manager.isBCRClaimed() == true) {
        var bcrModeCapability = manager.getBCRModeCapability();

        msg += ('BCRModeCapability : \n');
 
        for (key in bcrModeCapability) {
            msg += (' ' + key + ' - ' + bcrModeCapability[key] + '\n');
        }
    }
    else {
        msg += 'This function is usable after Claim.'
    }
}

setBCREventListener

Set the callback method which is related with barcode reader device.

syntax

setBCREventListener(eventName, javaScriptFunction);

Parameter

NameDescriptionObject typeOmission
eventName'pollingStatus' … Receives a status about polling of Claim management.String
javaScriptFunctionPerforms method which is related with eventName
*please refer to the following about the specification of a performed method
  • Receives a status about polling of Claim management.
    [Method]
      function(args)
    [Parameter]
      status … status
          Value: 3001 … It has passed the timeout about the polling of Claim management, therefore has worked internally same as "claim=false".
                    Example: it has kept to show the alert dialog, then has not performed the polling for Claim management from Browser.
                   3002 … It will internally work the same as "claim=false" after passing the timeout about polling of Claim management.
                    Exmaple: The html page has updated or moved the link during the polling of Claim management.   description … description of status
          Value: 'BarcodeReader claim automatically released!' (3001)
               'BarcodeReader claim automatically will release in
               the specified claimPollingTimout milliseconds!' (3002)
Object

Return value

DescriptionObject type
--
Example
<script type='text/javascript' src='js/StarWebPrintTrader.js'></script>
<script type='text/javascript' src='js/StarClaimManager.js'></script>
<script type='text/javascript'>
<!--
var manager = new StarClaimManager();

var pollingStatusListener = function(args) {
    try {
        if ((args.status != undefined) && (args.description != undefined)) {
            alert(args.status + ' : ' + args.description);
        }
    }
    catch (e) {
        alert(e.message);
    }
}

manager.setBCREventListener('pollingStatus', pollingStatusListener);
// -->
</script>
